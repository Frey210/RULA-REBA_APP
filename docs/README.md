# ErgoQuipt Documentation

Dokumen utama untuk perencanaan produk:

- `mvp_scope.md` - batas fitur MVP awal dan acceptance criteria.
- `priority_roadmap.md` - urutan implementasi prioritas.
- `future_features.md` - fitur post-MVP dan arah produk jangka panjang.

Dokumen teknis pendukung:

- `architecture.md` - arsitektur sistem edge, backend, database, dan desktop.
- `database_design.md` - rancangan tabel dan strategi data.
- `api_contract.md` - kontrak API dan payload antar komponen.
- `implementation_plan.md` - gate engineering dan rencana implementasi teknis.

Catatan brainstorming:

- `brainstorming-pengembangan.txt` - bahan diskusi awal; bukan dokumen keputusan final.
